# keepAlive 应用方案

该方案是结合当前焦点插件，对keepAlive某一个路由不同跳转状态下是否缓存的逻辑的处理。

# 应用场景描述

页面以 index=>list=>detail 顺序跳转时，index、list 两个页面需要缓存，detail 页面不需要； 如果页面从 list 返回到 index 要求清除 list 页面的缓存。

# 焦点插件配置逻辑

main.js 文件中 focus 配置逻辑

import Vue from "vue";
import $focus from "epg-focus-vue";
Vue.use($focus, { includeAttr: "index,list" });

# 入口模版文件配置逻辑

app.vue 入口文件中 keepalive 相关逻辑

// app.vue
<template>
	<router-view v-slot="{ Component }">
		<keep-alive :include="includeStr">
			<component :is="Component" />
		</keep-alive>
	</router-view>
</template>

<script setup>
import { keepAliveStore } from "focus-vue";
const includeStr = keepAliveStore().includeStr;
</script>

# 返回时清除缓存逻辑

返回时需要清除缓存的 list 页面处理逻辑

<script setup lang="ts" name="list">
import { keepAliveStore } from "focus-vue";
const { proxy } = getCurrentInstance() as ComponentInternalInstance;
proxy?.$Focus.addEvent({
	keyBack: () => {
		keepAliveStore().delPage("list");
	},
});
</script>
